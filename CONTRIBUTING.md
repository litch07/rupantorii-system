# Contributing Guide

Thanks for your interest in contributing to Rupantorii.

## Quick Start

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature`.
3. Make your changes and keep commits small and focused.
4. Run the app locally and validate key flows (catalog, cart, checkout, admin).
5. Open a pull request with a clear description.

## Development Guidelines

- Keep code readable and consistent with existing style.
- Avoid introducing new dependencies unless necessary.
- Use environment variables, never hardcode secrets.
- Prefer descriptive variable names and small functions.
- Update docs when behavior or setup changes.

## Suggested Tests

Manual verification checklist:
- Browse products and view details
- Add to cart and checkout
- Admin login and order workflow
- Reviews/questions submit and reply

## Reporting Issues

If you find a bug, open an issue and include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs or screenshots
